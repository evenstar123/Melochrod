# Harmony Engine Roadmap

最后更新：2026-03-09

## 当前基线

`v1` 已恢复为生产管线，`v2` 不再承担主链路职责。

当前工作重点不是继续补救 `v2`，而是：

1. 稳定 `v1` 生产效果与接口兼容性
2. 建立固定样例集和回归评测
3. 为未来 `v3` 研发准备可量化基准

## P0

1. 固化 `v1` 回归样例
2. 补齐生产接口测试：`/api/harmonize`、`/api/omr/harmonize`、流式 SSE
3. 清理仍带有旧架构叙事的前端与文档文案
4. 对云端部署做一次基线核对，确保线上与本地同为 `v1`

## P1

1. 建立标准曲库评测
2. 增加人工评分维度：调性稳定、终止感、风格贴合、可伴奏性
3. 提升 `v1` 的可观测性：缓存命中、RAG 命中、LLM 失败类型、输出异常统计

## P2

1. 评估哪些归档的 `v2` 能力值得旁路复用
2. 仅允许模块级实验，不允许整套新架构直接接管生产
3. 为 `v3` 先写评测门槛，再谈实现路线

## 明确不做

1. 不再把 `v2` 当作当前生产方向继续迭代
2. 不在缺少 benchmark 的情况下重启大规模架构替换
3. 不允许“理论更先进”作为替换 `v1` 的充分理由
